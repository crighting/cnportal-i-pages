# cnportal-i-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常以静态形式呈现，适用于快速浏览、功能演示或内容展示场景。

## 项目简介

cnportal-i-pages 是一个通用型页面归档仓库。仓库内收录的 HTML 页面不针对任何单一域名或具体网站，而是作为独立的、可复用的前端素材库存在。每个页面均保持轻量、自包含的设计，便于直接部署或嵌入其他项目。

## 目录说明

```
.
├── pages/               # 存放所有 HTML 页面文件
│   ├── example1.html    # 示例页面 1
│   └── example2.html    # 示例页面 2
├── assets/              # 公共资源目录（CSS、JS、图片等）
├── index.html           # 仓库根目录索引页（可选）
└── README.md            # 本文件
```

## 页面归档说明

- 所有页面均以 `.html` 文件形式存放于 `pages/` 目录下。
- 每个页面应保持独立，尽量不依赖外部资源，如需依赖公共资源请统一放置在 `assets/` 目录中。
- 页面命名建议使用小写字母与连字符，如 `example-page.html`。
- 提交新页面时，请确保文件编码为 UTF-8，无 BOM 头。

## 维护说明

- 欢迎提交新页面或对现有页面进行优化。
- 提交 Pull Request 前请确保页面在主流浏览器中正常显示。
- 请勿在页面中添加跟踪代码、广告或恶意脚本。
- 如有公共资源更新，请同步更新引用路径。
- 本仓库为纯前端静态内容，不涉及后端逻辑或数据库。

## 使用建议

你可以通过 GitHub Pages 或其他静态托管服务直接部署本仓库，从而在线访问所有归档页面。部署时请确保 `pages/` 目录结构完整。
